Internet

Klient-server-modellen

Klient-server-modellen innebär att klienten, till exempel din webläsare skickar en beställning (request) till en server och efterfrågar information som sedan visas som en websida.

När man pratar om server och servrar så kan det vara lätt att tänka på högljudda serverparker med många datorer i stora kylda lokaler, och så kan det ofta se ut, men själva servern är bara programvaran som körs på dessa datorer. Vilken dator som helst kan egentligen vara en server, så länge den har en serverprogramvara installerad.

Internetkommunikation

IP - Internet Protocol

Varje dator på internet får en tilldelad IP-adress, lite som ett telefonnummer, men vi människor är inga höjdare på att hålla massor av IP-adresser i minnet så därför anväder vi webadresser i form av domän-namn.toppdomän där domän-namnet till exempel är youtube och toppdomänen är .com.

DNS - Domain Name System

Kan liknas vid en telefonkatalog som används för att tolka en IP-adress 142.250.74.46 till en adress som vi människor kan förstå https://www.youtube.com.

Protokoll

För att all kommunikation på internet ska fungera så måste alla datorer använda samma uppsättning regler, det är detta regelverk som kallas protokoll.

Det finns många olika protokoll som används av olika tjänster, men de mest grundläggande är TCP som står för Transmission Control Protocol och IP som står för Internet Protocol, tillsammas bildar de TCP/IP som utgör två delar i fyra-lagersmodellen (applikation-, transport-, internet- och länklagret) där TCP (som arbetar i transportlagret) ansvarar för att alla små paket som skickas kommer fram och IP (i internetlagret) ser till att paketen hittar till rätt dator via IP-adresserna.

Länklagret är det fysiska lagret och utgörs av de faktiska sladdarna, switchar och routrar som finns i ditt och andras nätverk.

Applikationslagret är där protokoll som till exempel http, ftp, IMAP och POP återfinns, IMAP och POP är protokoll som används när man skickar och tar emot email.

HTTP & HTTPS

HTTP - Hypertext Transfer Protocol

är ett kommunikationsprotkoll som används för att överföra data för websidor (jämför html - Hypertext Markup Language) mellan klient och server.

HTTPS - Hypertext Transfer Protocol Secure

är ett kommunikationsprotkoll för krypterad transport av http protokollet och ska skydda mot så kallad man-in-the-middle attack där en obehörig tredje part kan läsa av trafiken mellan klient och server, därför används protokollet bland annat för betalningsöverföringar och annan känslig trafik.

HTTPS kräver att servern har ett digitalt certifikat som är signerat av en betrodd tredje part och man kan se att trafiken till en server är krypterad genom att det visas ett låst hänglås i adressfältet.

FTP, FTPS & SFTP

FTP - File Transfer Protocol

är ett kommunikationsprotkoll som används för filöverföring mellan en klient och en server / serverprogramvara. När man ska publicera sin websida på ett webhotell kan man med fördel använda sig av FTP.

FTPS - SSL File Transfer Protocol

är ett sätt att skicka filer krypterat, det använder sig av SSL Secure Socket Layer, det ska dock inte förväxlas med SFTP som använder sig av SSH Secure Shell.
FTPS är FTP med ett påbyggt lager av säkerhet, inga direkta standarder, använder två portar vid överföring en för handskakning och kontroll av certifikat och en för datatrafiken.

SFTP - SSH File Transfer Protocol / Secure File Transfer Protocol

är ett sätt att skicka filer krypterat, det använder sig av SSH Secure Shell, SFTP är en utveckling ur SSH för att kunna skicka filer säkert, SFTP har tydliga standarder.
SFTP använder sig bara av en port för all trafik.
